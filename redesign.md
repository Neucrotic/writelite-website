# redesign.md — WriteLite Website Redesign

Working file for the website redesign. Details are planned here; settled
decisions get lifted into CLAUDE.md's Project Specifics. This file is cleared
out as the work is implemented — retire it when the redesign ships.

We work item by item. Documentation is last.

---

## Item 1 — About section layout (landing page)

**Goal:** Rework the about section so the content skims easily without
overwhelming, and the dead space around it is put to use.

**Current thinking:** The columns feel awkward, but the current two- and
three-column looks aren't right either. The layout needs rethinking so a visitor
can skim everything at a glance. There's also empty space next to the screenshot
and under the bullet points that could be filled — a logo, or the "The Premise"
part of the about section, are candidates. Possible this is overkill and the real
answer is shipping and gathering user feedback.

**Open questions:**
- What replaces the column layout
- What fills the empty space — logo, "The Premise", or something else
- Redesign now vs ship and let real feedback drive it

---

## Item 2 — Donation page funding trackers

**Goal:** Two graphics on the donation page that show progress toward concrete
funding goals.

**Current thinking:** One tracker for the money needed to fund a month of
WriteLite development. A second for the $2,000 needed to afford a macOS release.

**Open questions:**
- Monthly development funding target amount
- How progress updates — manual, or driven from Stripe
- Visual form of the tracker

---

## Item 3 — Documentation page

*Do this last.*

**Goal:** A basic "how to use it" page — enough to make users comfortable,
without bloat or gaps.

**Current thinking:** Documentation needs its own dedicated pass so it carries
the essentials and nothing useless. The basics of how to use the app are enough
for now. The critical material is how `.wlite` files work — so users are
comfortable knowing where their work lives and how easily they can back
everything up.

**Open questions:**
- What counts as "the basics" for the first version
- Format and placement on the site
